How to use In-Remedy

In-Remedy is a remediation investigation workspace for Intune proactive remediation exports. The goal is simple: import one or more Intune files, focus the table on the columns you care about, filter the results, and export the final view.

Typical workflow

  1. 1
    Import your files

    Use the import area at the top of the main screen. You can drag and drop or choose one or more CSV or ZIP files manually, then click Import.

  2. 2
    Wait for the notification

    The app uploads and processes files in the background. Status appears as a toast notification. If you need to copy an error message, hover over the notification to keep it visible.

  3. 3
    Review the remediation summary

    The summary cards show high-level remediation and detection outcomes for the imported data. Use them as a quick overview before drilling into the table.

  4. 4
    Shape the table

    Use the Columns panel on the left to activate or deactivate fields. Expand only the column groups you need so the table stays focused and readable.

  5. 5
    Investigate and export

    Resize columns, reorder them from the table header drag handle, filter by values, then export the visible result set with Export CSV.

Supported files

.csv .zip

What ZIP import does

If a ZIP contains multiple remediation export CSVs, each CSV is extracted and processed as its own import batch.

What to expect

  • Large files may take a little time to process after upload.
  • The results table refreshes after processing completes.
  • If an import produces validation issues, the notification text can be copied.

Main screen areas

Import bar

  • Select or drop files.
  • Start imports.
  • Clear the database when you want a fresh session.

Columns panel

  • Grouped by Identity, Status, Detection, Remediation, and Timing & context.
  • Use active and inactive states to control visible fields.
  • Collapse unused groups to keep the sidebar tidy.

Results grid

  • Filter by text or multi-select dropdowns.
  • Resize and reorder columns.
  • Use the floating horizontal scrollbar for wide tables.

Useful notes

Hover notifications to keep them open Use Clear Database only when you want a fresh import session
  • The app is designed around remediation analysis rather than device inventory management.
  • Only the columns you activate are shown in the table and included in export.
  • The API currently starts with an empty database on each run, so restart means import again.
  • For the cleanest workflow, import first, then shape the table, then export.
